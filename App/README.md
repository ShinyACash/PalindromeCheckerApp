# PalindromeCheckerApp
Use Case 2: Hardcoded Palindrome Check
At this stage, the application moves beyond a simple startup and implements its first bit of core logic. The program now verifies if a predefined string is a palindrome.

The application now:

- Initializes: Starts from the main() method and displays versioning info.

- Processes: Defines a hardcoded String literal in the stack.

- Evaluates: Utilizes StringBuilder to reverse the string and compares it to the original.

- Reports: Outputs whether the string is a palindrome or not before exiting.

Current Logic Flow
1. START 
2. Print Welcome/Version
3. Load hardcoded String "madam"
4. Reverse String via StringBuilder
5. Compare original vs reversed
6. Print Result
7. EXIT
